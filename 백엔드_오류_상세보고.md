### 오류 #1: 에이전트 발행 실패

#### 📌 기본 정보
- **심각도**: 🔴 Critical
- **상태**: 미해결
- **영향 범위**: 에이전트 생성 기능 전체 불가

#### 🐛 에러 메시지
```
TypeError: 'parameters' is an invalid keyword argument for Agent
```

#### 📍 발생 위치
- **파일**: `backend/services/ai_hub/db/agent_repo.py`
- **라인**: 14
- **함수**: Agent 모델 생성 부분

#### 🔄 재현 방법
1. 프론트엔드에서 로그인
2. 채팅 페이지(`/chat`)로 이동
3. AI와 대화 진행 (최소 2~3턴)
4. 우측 상단 "에이전트 생성" 버튼 클릭
5. Step 1: 이름, 설명, 지침 확인/수정
6. Step 2: 카테고리, 공개 범위 선택
7. "완료" 버튼 클릭
8. **에러 발생**: 백엔드에서 `TypeError` 발생

#### 📊 API 호출 흐름
```
프론트엔드 → 백엔드

1. POST /agents/draft
   Body: { "selected_messages": [...] }
   ✅ 성공
   
2. PUT /agents/draft/step1
   Body: { "draft_id": "...", "name": "...", "description": "...", "instructions": "..." }
   ✅ 성공
   
3. PUT /agents/draft/step2
   Body: { "draft_id": "...", "category": "업무보조", "visibility": "private", "tags": [] }
   ✅ 성공
   
4. POST /agents/publish
   Body: { "draft_id": "..." }
   ❌ 실패 - TypeError 발생

#### 🔍 원인 분석
- `agent_repo.py:14`에서 Agent 모델 인스턴스 생성 시 `parameters`라는 키워드 인자를 전달
- Agent 모델이 `parameters` 인자를 받지 않음
- 가능한 원인:
  1. 모델 정의와 실제 사용 코드 간 불일치
  2. 잘못된 필드명 사용 (예: `parameters` → `config` 또는 다른 이름)
  3. 모델 마이그레이션 누락

#### 💡 해결 방안
1. `agent_repo.py:14` 코드 확인
2. Agent 모델 정의 확인 (`models.py` 또는 스키마 파일)
3. `parameters` 필드명 수정 또는 제거
4. 필요 시 DB 마이그레이션 실행

{

  ]
}
```

#### 📍 발생 위치
- **파일**: `backend/api/drive.py`
- **엔드포인트**: `POST /drive/documents`
- **계층**: API Router (Request Validation)

#### 🔄 재현 방법
1. 프론트엔드에서 로그인
2. 드라이브 페이지(`/drive`)로 이동
3. "업로드" 버튼 클릭

#### 📊 API 호출 정보

**프론트엔드 요청** (이미 수정 완료):
```
POST /drive/documents
Headers:
  Authorization: Bearer {access_token}
  Content-Type: multipart/form-data

FormData:
  file: File (실제 파일 객체)
  name: "example.pdf" (파일명)
  creator_id: "user123" (localStorage에서 읽음)
  creator_department: "개발팀" (localStorage에서 읽음)
  description: "" (빈 문자열 또는 사용자 입력)
  tags: "[]" (JSON.stringify([]))
```

**백엔드 응답**:
```
422 Unprocessable Entity
{
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "creator_id"],
      "msg": "Field required"
    }
  ]
}
```

#### 🔍 원인 분석
- 프론트엔드는 모든 필수 필드를 전송 중
- 백엔드에서 `creator_id`를 인식하지 못함
- 가능한 원인:
  1. FormData 파싱 문제 (multipart/form-data)
  2. Pydantic 스키마 정의 문제
  3. 필드명 불일치 (camelCase vs snake_case)
  4. 파일 업로드 핸들러 설정 문제

#### 💡 해결 방안

**확인 사항**:
1. `backend/api/drive.py`의 `upload_document` 엔드포인트 확인
2. Request 스키마 확인 (Pydantic 모델)
3. FormData 파싱 로직 확인

```typescript
// frontend/lib/api.ts
public async uploadDocument(file: File): Promise<Document> {
    const formData = new FormData();
    formData.append('file', file);
    formData.append('name', file.name);
    formData.append('creator_id', localStorage.getItem('user_id') || '');
    formData.append('creator_department', localStorage.getItem('department') || '');
    formData.append('description', '');
    formData.append('tags', JSON.stringify([]));

    return this.request<Document>('/drive/documents', {
        method: 'POST',
        body: formData,
        // Content-Type 헤더는 브라우저가 자동 설정
    });
}
```
