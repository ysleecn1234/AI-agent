# 🐳 도커(Docker)가 뭔가요? (쉽게 알아보기)

안녕! 우리가 만든 AI 프로그램을 실행하려면 **Docker(도커)**라는 친구가 꼭 필요해요.
도커가 무엇인지, 초등학생도 이해할 수 있게 설명해 줄게요!

---

## 1. 🍱 도커는 "마법의 도시락"이에요!

우리가 만든 프로그램을 다른 컴퓨터(친구 집, 학교 컴퓨터)에서 실행하려고 하면, 가끔 에러가 나면서 실행이 안 될 때가 있어요.
*"어? 내 컴퓨터에서는 잘 되는데 왜 네 컴퓨터에서는 안 되지?"*

이건 컴퓨터마다 설치된 프로그램(재료)이 달라서 그래요.
그래서 우리는 **"마법의 도시락(Container)"**을 만들었어요!

*   **도커(Docker):** 이 도시락을 만들어주고 배달해주는 서비스예요.
*   **컨테이너(Container):** 밥, 반찬, 숟가락까지 모든 게 다 들어있는 **완성된 도시락**이에요.
    *   이 도시락만 있으면, 어떤 컴퓨터를 쓰든 **똑같은 맛(똑같은 실행 결과)**을 낼 수 있어요!

---

## 2. 📜 레시피(Image)와 도시락(Container)

도커에는 두 가지 중요한 단어가 있어요.

1.  **이미지 (Image) = 요리 레시피 📜**
    *   *"파이썬 3.11을 깔고, FastAPI를 설치하고, 코드를 복사해라"* 라고 적힌 설명서예요.
    *   이 설명서는 읽기만 할 수 있고 수정할 수는 없어요.

2.  **컨테이너 (Container) = 실제 도시락 🍱**
    *   레시피(Image)를 보고 진짜로 만들어낸 도시락이에요.
    *   우리가 서버를 켠다는 건, 이 도시락 뚜껑을 열어서 먹는(실행하는) 것과 같아요.

---

## 3. 🏗️ 우리 도시락 구성 (`docker-compose.yml`)

터미널에 `docker-compose up`을 치면, 도커가 **4단 도시락**을 뚝딱 차려줍니다.

### 🍱 1층: 메인 밥 (Backend App)
*   **이름:** `in7-backend`
*   **하는 일:** 우리가 만든 파이썬 프로그램, 웹 서버.
*   **특징:** 사용자의 주문(요청)을 받아서 처리해요.

### 🥩 2층: 고기 반찬 (PostgreSQL)
*   **이름:** `in7-postgres`
*   **하는 일:** 회원 정보, 에이전트 설정, 대화 내용을 저장하는 **금고**.
*   **특징:** 서버가 꺼져도 데이터는 사라지지 않아요.

### ⚡️ 3층: 김치 (Redis)
*   **이름:** `in7-redis`
*   **하는 일:** 자주 쓰는 정보를 잠깐 적어두는 **메모장**.
*   **특징:** 엄청 빠르지만, 전원이 꺼지면 지워질 수 있어요.

### 🤖 4층: AI 비밀 반찬 (Milvus)
*   **이름:** `milvus-standalone`
*   **하는 일:** AI가 읽은 책(문서) 내용을 저장하는 **AI 도서관**.
*   **특징:** "비슷한 내용 찾아줘"라고 물어보면 0.1초 만에 찾아줘요.

---

## 4. 🚀 어떻게 실행하나요?

터미널에 딱 한 줄만 입력하면 도커가 이 4단 도시락을 뚝딱 만들어줘요.

```bash
./deploy.sh
```

이 명령어를 치면:
1.  새로운 코드를 가져오고 (**장보기**)
2.  도시락을 만들고 (**요리하기**)
3.  짠! 하고 실행해줍니다 (**상 차리기**)

---

## 5. 🔬 우리 도시락 깊게 파헤치기 (상세 분석)

`docker-compose.yml` 파일 안에 적힌 **외계어 같은 설정들**, 궁금하시죠?
하나씩 뜯어보면 어렵지 않아요!

### 📋 서비스 목록 (Services)

| 서비스 이름 | 포트(Port) | 역할 설명 | 의존성 (Depends On) |
| :--- | :--- | :--- | :--- |
| **backend** | `8000` | **[주인공]** 웹 서버 (FastAPI). 우리가 코드를 짜는 곳이에요. | postgres, redis, milvus |
| **postgres** | `5432` | **[RDB]** 관계형 DB. 사용자, 에이전트 정보를 엑셀 표처럼 저장해요. | - |
| **redis** | `6379` | **[Cache]** 캐시 서버. 로그인 세션이나 임시 데이터를 저장해요. | - |
| **milvus** | `19530` | **[Vector DB]** 벡터 DB. 문장을 숫자로 바꿔서 저장해요 (AI 검색용). | etcd, minio |
| *etcd* | - | (조연) Milvus가 데이터를 찾을 때 쓰는 '색인표' 역할을 해요. | - |
| *minio* | `9001` | (조연) Milvus가 데이터를 저장하는 '하드디스크' 역할을 해요. | - |

### 🔗 연결 고리 (Networks & Volumes)

*   **포트(Ports):**
    *   `8000:8000`: 내 컴퓨터의 8000번 문으로 들어가면, 컨테이너의 8000번 방으로 연결해줘요.
    *   그래서 우리가 인터넷 주소창에 `localhost:8000`을 치면 서버에 접속되는 거예요!

*   **볼륨(Volumes):**
    *   컨테이너(도시락)는 다 먹고 버리면(삭제하면) 안에 내용물도 사라져요.
    *   하지만 **데이터(DB)**는 사라지면 안 되잖아요?
    *   그래서 **"볼륨"**이라는 튼튼한 외장 하드를 연결해 둡니다.
    *   `postgres_data`, `milvus_data` 등이 바로 그 외장 하드예요. 도시락 통을 바꿔도 데이터는 안전해요!
