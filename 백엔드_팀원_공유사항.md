# 백엔드 팀원 공유사항 (2026.02.15)

## 📢 중요 공지

프론트엔드 개발이 완료되었으나, **백엔드 버그로 인해 일부 기능이 동작하지 않습니다.**
아래 버그 수정 후 전체 기능 테스트가 필요합니다.

---

## 🔴 긴급 수정 필요 (Critical)

### 1. 에이전트 발행 실패
**에러 메시지**:
```
TypeError: 'parameters' is an invalid keyword argument for Agent
```

**발생 위치**: `backend/services/ai_hub/db/agent_repo.py:14`

**재현 방법**:
1. 프론트엔드에서 채팅 진행
2. "에이전트 생성" 버튼 클릭
3. 이름, 설명 등 입력 후 "완료" 클릭
4. 백엔드에서 에러 발생

**원인**: Agent 모델 생성 시 `parameters` 키워드 인자가 잘못 전달됨

**영향**: 에이전트 생성 기능 전체 불가

**프론트엔드 API 호출 순서**:
```
POST /agents/draft (selected_messages 전송)
  ↓
PUT /agents/draft/step1 (name, description, instructions)
  ↓
PUT /agents/draft/step2 (category, visibility, tags)
  ↓
POST /agents/publish ← 여기서 에러 발생
```

---

### 2. 드라이브 문서 목록 조회 실패
**에러 메시지**:
```json
{
  "detail": "문서 목록 조회 실패: PostgresClient.list_documents() got an unexpected keyword argument 'department'"
}
```

**발생 위치**: `backend/api/drive.py` → `drive_service.list_documents()`

**재현 방법**:
1. 프론트엔드에서 `/drive` 페이지 접속
2. 문서 목록 로딩 시도
3. 500 에러 발생

**원인**: 
- API 라우터에서 `department` 파라미터를 전달
- `PostgresClient.list_documents()` 메서드가 해당 파라미터를 받지 못함

**영향**: 
- 드라이브 문서 목록 조회 불가
- 문서 미리보기 불가
- 문서 다운로드 불가

**해결 방안**:
1. `PostgresClient.list_documents()` 메서드에 `department` 파라미터 추가
2. 또는 API 라우터에서 `department` 파라미터 제거

---

### 3. 드라이브 문서 업로드 실패
**에러 메시지**:
```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "creator_id"],
      "msg": "Field required",
      "input": null
    }
  ]
}
```

**발생 위치**: `backend/api/drive.py` → `upload_document`

**재현 방법**:
1. 프론트엔드에서 `/drive` 페이지 접속
2. "업로드" 버튼 클릭
3. 파일 선택 후 업로드 시도
4. 422 에러 발생

**프론트엔드에서 전송하는 데이터** (이미 수정 완료):
```javascript
formData.append('file', file);
formData.append('name', file.name);
formData.append('creator_id', localStorage.getItem('user_id') || '');
formData.append('creator_department', localStorage.getItem('department') || '');
formData.append('description', '');
formData.append('tags', JSON.stringify([]));
```

**현재 상태**: 
- 프론트엔드는 모든 필수 필드를 전송 중
- 백엔드에서 422/500 에러 발생 (정확한 원인 미상)

**확인 필요 사항**:
1. `upload_document` 엔드포인트의 요청 스키마 확인
2. `creator_id`, `creator_department` 필드 처리 로직 확인
3. 백엔드 로그에서 상세 에러 확인

---

### 4. Milvus 벡터 검색 실패 (Medium)
**에러 메시지**:
```
MilvusException: Unsupported field type: 0
```

**발생 위치**: Milvus 벡터 검색 관련 코드

**재현 방법**: 
- 벡터 검색 기능 사용 시 발생 (정확한 재현 경로 미확인)

**원인**: Milvus 컬렉션 스키마 문제로 추정

**영향**: 벡터 기반 검색 기능 동작 불가

**우선순위**: 중간 (다른 버그 수정 후 처리)

## ✅ 오늘 완료된 프론트엔드 작업

### 1. 채팅 기능 개선
#### 마크다운 렌더링
- `react-markdown` 라이브러리 사용
- AI 답변에만 적용 (사용자 메시지는 일반 텍스트)
- 제목, 리스트, 코드 블록, 인용구 등 스타일링
- **파일**: `frontend/app/chat/page.tsx`

#### 세션 관리
- 백엔드에서 반환하는 `session_id` 저장
- 다음 메시지 전송 시 `session_id` 포함
- 대화 컨텍스트 유지
- **파일**: `frontend/app/chat/page.tsx`

#### 에이전트 실시간 추천
- 입력창에 500ms 디바운스 적용
- 3자 이상 입력 시 자동 추천
- `GET /agents/recommend?query={message}` 호출
- **파일**: `frontend/app/chat/page.tsx`

### 2. 에이전트 생성 개선
#### 다단계 API 연동
백엔드의 다단계 프로세스에 맞춰 프론트엔드 수정:
```
1. POST /agents/draft
   - Body: { selected_messages: [...] }
   
2. PUT /agents/draft/step1
   - Body: { name, description, instructions }
   
3. PUT /agents/draft/step2
   - Body: { category, visibility, tags }
   
4. POST /agents/publish
   - Body: { draft_id }
```

#### 메타데이터 생성
- `/generate/agent-metadata` 엔드포인트 미구현으로 인한 404 에러 해결
- 프론트엔드에서 간단한 키워드 기반 추론 로직 구현
- **파일**: `frontend/components/chat-action-modals.tsx`, `frontend/app/agents/create/page.tsx`

### 3. 드라이브 기능 추가
#### 문서 다운로드
- 문서 목록의 `...` 메뉴에 "다운로드" 옵션 추가
- `GET /drive/documents/{doc_id}/file?download=true` 호출
- Blob 다운로드 및 자동 파일명 설정
- **파일**: `frontend/app/drive/page.tsx`

#### 문서 미리보기 (iframe)
- **문제**: iframe은 Authorization 헤더를 전송할 수 없어 401 에러 발생
- **해결**: 
  1. fetch로 파일을 Blob으로 다운로드 (Authorization 헤더 포함)
  2. `URL.createObjectURL()`로 Blob URL 생성
  3. iframe src에 Blob URL 설정
- **API**: `GET /drive/documents/{doc_id}/file`
- **파일**: `frontend/app/drive/documents/[id]/page.tsx`

#### 문서 업로드 개선
- `creator_id`, `creator_department`, `description`, `tags` 필드 추가
- localStorage에서 사용자 정보 읽어서 FormData에 포함
- **파일**: `frontend/lib/api.ts`

### 4. 인증 시스템 개선
#### 401 에러 자동 처리
- API 요청 시 401 Unauthorized 발생하면:
  1. localStorage에서 `access_token` 삭제
  2. `/auth/login` 페이지로 자동 리다이렉트
  3. 단, 이미 auth 페이지에 있으면 리다이렉트 안 함
- **파일**: `frontend/lib/api.ts`

#### 토큰 관리 개선
- 생성자에서 토큰을 저장하지 않음
- 매 요청마다 localStorage에서 최신 토큰 읽기
- 토큰 갱신 시 즉시 반영
- **파일**: `frontend/lib/api.ts`

### 5. 기타 개선
- **브랜딩 변경**: 모든 페이지에서 "IN7" → "ISOR"
- **하드코딩 제거**: localhost 주소를 `process.env.NEXT_PUBLIC_API_URL` 또는 서버 IP로 변경

### 버그 수정 후 테스트 시나리오

#### 1. 에이전트 생성 테스트
1. 프론트엔드에서 로그인
2. 채팅 페이지로 이동
3. AI와 대화 (최소 2~3턴)
4. "에이전트 생성" 버튼 클릭
5. 이름, 설명 등 확인/수정
6. "완료" 버튼 클릭
7. **예상 결과**: 에이전트가 성공적으로 생성되고 목록에 표시

#### 2. 드라이브 업로드 테스트
1. 프론트엔드에서 로그인
2. 드라이브 페이지로 이동
3. "업로드" 버튼 클릭
4. 파일 선택 (PDF, DOCX 등)
5. 업로드 실행
6. **예상 결과**: 파일이 업로드되고 목록에 표시

#### 3. 드라이브 목록/미리보기 테스트
1. 드라이브 페이지 접속
2. **예상 결과**: 문서 목록이 표시됨
3. 문서 클릭
4. **예상 결과**: 문서 미리보기 iframe이 표시됨
5. `...` 메뉴에서 "다운로드" 클릭
6. **예상 결과**: 파일이 다운로드됨

#### 4. 에이전트 추천 테스트
1. 채팅 페이지 접속
2. 입력창에 "마케팅" 입력
3. 0.5초 대기
4. **예상 결과**: 마케팅 관련 추천 에이전트가 표시됨
